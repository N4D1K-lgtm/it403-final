The Path to Remediation
OOP Sin Practical Remedies and Insights
Envy (Feature Envy) Instead of reaching into other classes, let classes communicate via methods. Think 'Ask, don't tell' to maintain encapsulation.
Greed (Unnecessary Encapsulation) Keep it simple. Don't make it difficult to access data that you actually need to be able to access.
Lust (Premature Optimization) First, make it work; then make it fast. Optimize based on evidence (profiling), not assumptions.
Wrath (Pattern Abuse) Patterns are tools, not goals. Use them when they simplify, not complicate, your design.
Pride (Goliath Objects) Split mammoth classes into cohesive units. Each class should have a single purpose or responsibility.
Gluttony (Excessive Inheritance) Prefer composition over inheritance. Use interfaces for polymorphism. Remember, not every relationship is a 'is-a'.
Sloth (Type Indolence) Define clear roles for each class. Avoid 'utility' or 'manager' classes that become dumping grounds for unrelated functionality.
